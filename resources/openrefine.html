Common GREL Recipes

Curated examples:

  1. Cleaning text:

    value.trim()                    # Remove whitespace
value.toLowercase()             # Convert to lowercase
value.toTitlecase()             # Title case
value.replace("old", "new")     # Replace text

  2. Working with dates:

    value.toDate()                          # Parse date
value.toDate().toString("yyyy-MM-dd")   # Format date

  3. Conditional logic:

    if(value == "N/A", null, value)         # Replace N/A with null
if(value.length() > 10, "long", "short") # Categorize by length

  4. Splitting text:

    value.split(",")                # Split by comma
value.split(",")[0]             # Get first element

More recipes: https://github.com/OpenRefine/OpenRefine/wiki/Recipes

Troubleshooting & Tips

Common Issues

WarningFrequent Problems & Solutions

Problem: OpenRefine won‚Äôt start - Solution: Check if port 3333 is in use, restart computer

Problem: Large files crash OpenRefine - Solution: Increase memory allocation (see documentation)

Problem: Can‚Äôt undo after closing project - Solution: Export operations before closing, reimport if needed

Problem: Lost work - Solution: OpenRefine auto-saves, but export projects regularly

Performance Tips

  • Filter before transforming: Narrow down data first
  • Use facets efficiently: Close unused facets
  • Export regularly: Don‚Äôt rely on auto-save alone
  • Split large files: Process in chunks if very large

Regular Expressions in OpenRefine

Regex Resources

GREL supports regular expressions for advanced pattern matching.

Learn Regex

  • Interactive tutorial: https://regexone.com/
  • Regex tester: https://regex101.com/ (test patterns before using)
  • Cheat sheet: https://www.rexegg.com/regex-quickstart.html

Regex in OpenRefine

Common patterns:

# Match emails
value.match(/[\w.-]+@[\w.-]+\.\w+/)

# Extract numbers
value.match(/\d+/)

# Remove non-alphanumeric
value.replace(/[^a-zA-Z0-9]/, "")

# Find phone numbers (US format)
value.match(/\d{3}-\d{3}-\d{4}/)
