File Naming Conventions

Good file naming is fundamental to research data management. Here are comprehensive resources to help you develop and maintain consistent naming practices.

Key Principles

ImportantQuick Reference: File Naming Rules
  1. Use dates: YYYY-MM-DD format (ISO 8601)
  2. No spaces: Use hyphens - or underscores _
  3. Be descriptive: But keep it concise
  4. Version numbers: v01, v02, not ‚Äúfinal‚Äù or ‚Äúfinal2‚Äù
  5. Avoid special characters: No √©, √†, /, , ?, *, etc.
  6. Lowercase: For consistency (optional but recommended)

Example: 2026-02-09_interview_participant-05_v02.docx

Backup Strategies: The 3-2-1 Rule

Understanding 3-2-1

TipThe 3-2-1 Backup Rule
  • 3 copies of your data (original + 2 backups)
  • 2 different media types (e.g., local drive + cloud)
  • 1 copy off-site (cloud or physically elsewhere)

File Format Best Practices

Choosing Preservation-Friendly Formats

Long-term accessibility depends on choosing the right file formats.

Key Resources

  1. Recommended Formats Statement - Library of Congress

  2. File Formats for Preservation - Digital Curation Centre

Quick Reference

Data Type Preferred Formats Avoid
Text .txt, .pdf/A, .xml, .html .doc, .pages
Tabular .csv, .tsv, .xml .xlsx, .numbers
Images .tiff, .png, .jpg .psd, proprietary RAW
