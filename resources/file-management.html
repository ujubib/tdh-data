File Naming & Backup for Digital Humanities

Best Practices for SSH Research

Introduction

Systematic file management is essential for Digital Humanities research, especially for projects involving multiple sources, collaborators, or long-term data collection.

ImportantFor Doctoral Researchers

With projects spanning 3-5 years, involving multiple institutions, and generating diverse data types, consistent file naming and backup strategies prevent data loss and enable collaboration.

File Naming Conventions

Core Principles

TipQuick Reference: File Naming Rules
  1. Use dates: YYYY-MM-DD format (ISO 8601) - universal across countries
  2. No spaces: Use hyphens - or underscores _
  3. Be descriptive: Include project, source, version
  4. Version numbers: v01, v02, v03 (not “final”)
  5. No special characters: Avoid é, à, ç, ñ, /, , ?, *
  6. Language-neutral: Use English or transliterated names for international collaboration

Example for archival photos:
2025-06-15_paris-natlib_ms-fr-12345_fol-23r_v01.jpg
(Date_Archive_Manuscript_Folio_Version)

Naming Conventions for Specific DH Data Types

Transcriptions

[date]_[source-archive]_[document-id]_transcription_[version].txt

Example:
2025-06-20_bnf_ms-lat-8501_fol-12v_transcription_v03.txt

Digitized Images

[date]_[archive]_[collection]_[item-id]_[folio/page]_[version].jpg

Example:
2025-03-15_vat-lib_pal-lat_1071_fol-45r_v01.tif

Interview Recordings

[date]_interview_[participant-id]_[location]_[recorder].mp3

Example:
2025-09-10_interview_p05_rome_audio-01.mp3

3D Models

[date]_[site]_[object-type]_[scan-method]_[processing-stage].obj

Example:
2025-07-22_pompeii_fresco_photogrammetry_raw.obj
2025-07-22_pompeii_fresco_photogrammetry_cleaned.obj

Metadata Files

[date]_[project]_[data-type]_metadata_[version].csv

Example:
2025-11-05_choral-manuscripts_images_metadata_v02.csv

Backup Strategies: The 3-2-1 Rule

Understanding 3-2-1 for Research Projects

TipThe 3-2-1 Backup Rule
  • 3 copies of your data
    • Original working copy
    • Backup at institution
    • Backup at home OR cloud
  • 2 different media types
    • Local hard drive (laptop/desktop)
    • External hard drive OR university server
    • Cloud storage (Nextcloud, Huma-Num)
  • 1 copy off-site
    • Geographically separate
    • Protected from local disasters (fire, theft)
    • Accessible when traveling or working remotely

Practical Implementation

Challenge: Managing data across multiple work locations

Solution: 1. Primary working storage: University Nextcloud/cloud (syncs everywhere) 2. Local backup: External hard drive at office or home 3. Off-site backup: Institutional server or Huma-Num

Best practices: - Keep active projects on cloud storage for easy access - Carry external drive for large files (videos, high-res images) - Test restoration periodically to verify backups work

File Formats for Digital Humanities

Preservation-Friendly Formats by Data Type

NotePrioritize open, documented formats

Proprietary formats risk obsolescence. Choose formats with: - Open specifications - Wide tool support - No licensing restrictions - Good for long-term preservation

Data Type Preferred Formats Avoid Notes
Plain text .txt, .md .pages, .doc UTF-8 encoding!
Structured text .xml (TEI), .json Proprietary XML TEI for scholarly editions
Tabular data .csv, .tsv .xlsx, .numbers Use semicolons for European locales
Images (photos) .tiff (archival), .jpg (access) .psd, proprietary RAW Keep RAW separately
Images (diagrams) .svg, .png .ai, .sketch SVG is scalable
3D models .obj, .ply, .gltf .max, .blend Include textures separately
Audio .wav, .flac .mp3 (lossy) Lossless for primary sources
Video .mp4 (H.264), .mkv .mov, proprietary codecs Document settings
Databases .sql, .csv exports .mdb, .accdb Export to open formats
Manuscripts .pdf/A, TEI .xml .docx, .indd PDF/A for archival

The DH standard for encoding texts

  • Website: https://tei-c.org/

Data Management Plans (DMP)

Why DMPs Matter

Required by: - European Commission (Horizon Europe) - National funders (ANR, ERC) - Many doctoral schools

A DMP specifies: - What data you’ll collect/create - How it will be stored and backed up - Who can access it - How it will be preserved after your PhD - Costs and responsibilities

Metadata Best Practices for Cultural Heritage

Why Metadata Matters

Without metadata, your data is useless: - Where was this photo taken? - Who created this document? - What do these column headers mean? - Can I reuse this 3D model?

Metadata Standards for DH

Domain Standard Link
Cultural Objects CIDOC-CRM http://www.cidoc-crm.org/
Libraries Dublin Core, MARC https://www.dublincore.org/
Archives EAD, ISAD(G) https://www.loc.gov/ead/
Museums LIDO, SPECTRUM http://lido-schema.org/
Manuscripts TEI msDesc https://tei-c.org/release/doc/tei-p5-doc/en/html/MS.html
Images IIIF https://iiif.io/
Geographic GeoJSON, KML https://geojson.org/

Minimal Metadata for Any File

Always include: - Creator/Author: Who made this? - Date: When (YYYY-MM-DD)? - Description: What is it? - Source: Where did it come from? - License/Rights: Can others use it?

For images, also include: - Archive/institution - Collection name - Shelfmark/identifier - Physical description
